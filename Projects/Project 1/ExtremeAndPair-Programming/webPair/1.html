Estas dos metodologías tienen muy pocas diferencias y una de esas puede ser la manera a la hora de trabajar, ya que el pair programming como el mismo nombre lo dice, es una metodología donde se trabaja en parejas y cada pareja en una misma computadora, esa es la diferencia más significativa. El extreme programming tiene un enfoque para trabajar en empresas muy pequeñas y su propósito es mejorar la calidad del software y estar preparados para recibir cambios constantemente.
