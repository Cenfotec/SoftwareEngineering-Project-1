Historia y Orígenes

JavaScript fue desarrollado originalmente por Brendan Eich de Netscape con el nombre de Mocha, el cual fue renombrado posteriormente a LiveScript. Tiempo más tarde, Eich decide optar por la ayuda mutua con Sun Microsystems, creadores del lenguaje de programación Java; entonces debido a cuestiones de mercadeo, Sun Microsystems y Brendan Eich deciden renombrar “Live Script” por “Java Script” debido a la gran popularidad de la palabra “Java”. En 1997, Netscape, la compañia donde trabajaba Brendan Eich, decide estandarizar Java Script para evitar problemas con otras compañías y tecnologías. Entonces se envía la especificación de Java Script v1.1 a European Computer Manufacturers Association, mejor conocido como ECMA y nace la estandarización con el nombre de ECMA-262, y el lenguaje de programación se define como ECMAScript.

Versiones

La primera versión de ECMAScript fué cuando se le entregó la especificación JavaScript 1.1 a ECMA o European Computer Manufacturers Association hasta que lo estandarizaron. Un año más tarde que se publica el estándar, se crea la version ECMAScript 2. La version de ECMAScript 4 nunca se publicó, y la razón por la cual nunca se distribuyó fue por problemas técnicos y por su gran complejidad. No fue hasta Diciembre del 2009 que se publicó la siguiente versión, ECMAScript 5. En el 2015 se publicó ECMAScript 6. Luego la versión ECMAScript 7, publicada en el 2016. Por último, en Junio del 2017, se distribuye la version ECMAScript 8.

Estándares

En 1997 el primer estándar de JavaScript es creado, el lenguaje de programación se define como ECMAScript, y European Computer Manufacturers Association decide llamarlo ECMA-262. Luego en 1998, con la publicación de ECMAScript 2, surgen algunas modificaciones para que al final conformen el estandar internacional ISO/IEC 16262. Años más tarde, en el 2005, Brendan Eich, diseñador de JavaScript junto a Mozilla con ayuda de ECMA International, trabajan en ECMAScript específicamente para XML o Lenguaje de Marcado Extensible, esto lleva a un nuevo estándar llamado “ECMA-357” Posteriormente se estandariza ECMAScript para XML (E4X) en ActionScript 3, la cual estaba basado en el ECMAScript 4 original En Junio del 2011 se publica ECMAScript 5.1 con la tercera edición del estándar internacional ISO/IEC 16262; y en Junio del 2017 declaran el estándar ECMA-402 cuarta edición. El estandar mas reciente es ECMA-262 edición número nueve la cual implementa el lenguaje de especificación para ECMAScript 2018.

¿Quién lo controla?

Una gran parte de Java Script es open source, porque es una implementación del estándar del lenguaje ECMAScript. Diseñado primeramente por Brendan Eich y conformado por los desarrolladores como Netscape Communications Corporation, Mozilla Foundation y ECMA International. Estos tres protagonistas se encargan de diferentes partes de Java Script.
